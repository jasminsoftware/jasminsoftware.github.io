Accounts Payable | Create Payment

Overview

api/{tenantKey}/{orgKey}/accountsPayable/processOpenItems
URL api/{tenantKey}/{orgKey}/accountsPayable/processOpenItems
Methods Supported GET, POST
Description Service to allow picking of opened item to process for payment purpose.

GET OPERATIONS

GET Create Payment

GET Create Payment

Returns a paginated list of available entities that can be processed by the Payments service.

The following elements are required:

pageIndex (int) The number of the page you want to consume the results.
pageSize (int) Specify how many items you want each page to return, up to 1000 items.
company (string) The company code is the central organizational unit within the system. This is a query string parameter.
documentDate (DateTime) The date when the document is issued. This is a query string parameter.
documentExchangeRate (decimal) The price of one country's currency expressed in another country's currency. In other words, the rate at which one currency can be exchanged for another. The default is infered using the defined exchange rates on the system for the document date and currency. This is a query string parameter.
party (string) The customer designated in the account posting as the party to whom should be sent for payment. It may or may not be the party to whom the goods or services are delivered. This is a query string parameter.
currency (string) Money unit used by the system, such as Euros (EUR) or United States Dollars (USD). This value is important for currency exchanges or to set the number of decimal places used to represent values. The default is defined on the customer party. This is a query string parameter.
documentType (string) Defines the account posting's purpose. Distinguishes the business transactions to be posted. Part of the document natural key. The default account posting type is defined in the sales setup configuration setting. This is a query string parameter.

The following elements are optional:

dueDateFrom (DateTime?) The initial date in which you can pay. This is a query string parameter.
dueDateUntil (DateTime?) Date in which the payment is due. In some countries it is also known as law date. This is a query string parameter.
paymentMethod (string) Financial transfer method used by the entity responsible for paying a transaction. For example, cash, transfer or check payments. The default is defined on the customer party. This is a query string parameter.
total (decimal?) Document total. Internally calculated. This is a query string parameter.
sourceDoc (string) The source document key. This is a query string parameter.

Response:

If successful, returns a list of resources available to process. See OpenAccountPostingLine.

Response status codes:

200 The operation was successful.

Resource URL:

api/{tenantKey}/{orgKey}/accountsPayable/processOpenItems/{pageIndex}/{pageSize}?company={company}&documentDate={documentDate}&documentExchangeRate={documentExchangeRate}&party={party}&currency={currency}&documentType={documentType}
GET /accountsPayable/processOpenItems/{pageIndex}/{pageSize}?company={company}&documentDate={documentDate}&documentExchangeRate={documentExchangeRate}&party={party}&currency={currency}&documentType={documentType}

POST OPERATIONS

POST Create Payment

POST Create Payment

Process the selected resources by the Payments service.

The following elements are required:

list (OpenAccountPostingLine) List of OpenAccountPostingLine to process.
See OpenAccountPostingLine.
companyKey (string) Company key to which the requested OpenAccountPostingLine is associated to.

Response:

If successful, returns an empty body response and the URL of the processed items in the response location header.

Response status codes:

201 The operation was successful.

Resource URL:

api/{tenantKey}/{orgKey}/accountsPayable/processOpenItems/{companyKey}

Resources


OpenAccountPosting

The following elements are required:

company (string) The company code is the central organizational unit within the system.
documentDate (DateTime) The date when the document is issued.
documentExchangeRate (decimal) The price of one country's currency expressed in another country's currency. In other words, the rate at which one currency can be exchanged for another. The default is infered using the defined exchange rates on the system for the document date and currency.
party (string) The customer designated in the account posting as the party to whom should be sent for payment. It may or may not be the party to whom the goods or services are delivered.
currency (string) Money unit used by the system, such as Euros (EUR) or United States Dollars (USD). This value is important for currency exchanges or to set the number of decimal places used to represent values. The default is defined on the customer party.
selectedCount (int) Number of lines selected. Internally calculated.
documentType (string) Defines the account posting's purpose. Distinguishes the business transactions to be posted. Part of the document natural key. The default account posting type is defined in the sales setup configuration setting.

The following elements are optional:

dueDateFrom (DateTime) The initial date in which you can pay.
dueDateUntil (DateTime) Date in which the payment is due. In some countries it is also known as law date.
paymentMethod (string) Financial transfer method used by the entity responsible for paying a transaction. For example, cash, transfer or check payments. The default is defined on the customer party.
total (decimal) Document total. Internally calculated.
sourceDoc (string) The source document key.


OpenAccountPostingLine

The following elements are required:

selected (bool) Indicates if the line has been selected.

The following elements are optional:

sourceDoc (string) The source document key.
dueDate (DateTime) Date in which the payment is due. In some countries it is also known as law date.
paymentMethod (string) Financial transfer method used by the entity responsible for paying a transaction. For example, cash, transfer or check payments. The default is defined on the customer party.
amount (decimal) The total line amount.
open (decimal) The remain amount to be settled.
settled (decimal) The settled amount.
discount (decimal) The line item discount. The default is 0.
withholded (decimal) The line withholded amount.
accountPostingId (Guid) The account posting id for the line.
exchangeRate (decimal) The price of one country's currency expressed in another country's currency. In other words, the rate at which one currency can be exchanged for another. The default is inferred using the defined exchange rates on the system for the document date and currency.
natureId (Guid) The document line nature id. It can be Credit or Debit.
sourceDocId (Guid) The source document key.
