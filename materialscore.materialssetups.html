Materials Management | Materials Setup

Overview

URL api/{tenantKey}/{orgKey}/materialsCore/materialsSetups
Methods Supported GET, PUT
Description Service to manage entities of type 'MaterialsSetup'.

GET OPERATIONS

GET MaterialsSetups

Returns all Materials Setup user has access to.

Response:

If successful, this method returns a list of Materials Setup. See MaterialsSetupResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /materialsCore/materialsSetups

GET MaterialsSetups (paged)

Returns a paginated list of Materials Setup user has access to.

The following elements are optional:

page (int) The number of the page you want to consume the results.
pageSize (int) Specify how many items you want each page to return, up to 1000 items.

Response:

If successful, this method returns a list of Materials Setup. See MaterialsSetupResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /materialsCore/materialsSetups

GET MaterialsSetup by identifier

Returns information for a given Materials Setup, by specifying its identifier.

The following elements are required:

id (string) The identifier for the Materials Setup.

Response:

If successful, returns a resource of the Materials Setup. See MaterialsSetupResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /materialsCore/materialsSetups/{id}

GET MaterialsSetup by key

Returns information for a given Materials Setup, by specifying its key.

The following elements are required:

company (string) The company for the Materials Setup.

Response:

If successful, returns a resource of the Materials Setup. See MaterialsSetupResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /materialsCore/materialsSetups/{company}

GET Print MaterialsSetup by identifier

Prints the specified MaterialsSetup.

The following elements are required:

id (string) The identifier for the MaterialsSetup.

The following elements are optional:

template (string) The report template.

Response:

If successful, returns a stream with the file of the printed MaterialsSetup.

Response status codes:

200 The operation was successful.

Resource URL:

GET /materialsCore/materialsSetups/{id}/print

GET Print MaterialsSetup by key

Prints the specified MaterialsSetup by specifying its key.

The following elements are required:

company (string) The company for the Materials Setup.

The following elements are optional:

template (string) The report template.

Response:

If successful, returns a stream with the file of the printed MaterialsSetup.

Response status codes:

200 The operation was successful.

Resource URL:

GET /materialsCore/materialsSetups/{company}/print

PUT OPERATIONS

PUT Set Company MaterialsSetup

Updates the Company for the specified Materials Setup.

The following elements are required:

company (string) The company for the Materials Setup.
value (string) In the request body, supply the new value for the Company.

Response:

If successful, this method returns an empty response body.

Response status codes:

200 The operation was successful.

Resource URL:

PUT /materialsCore/materialsSetups/{company}/company

PUT Set IsActive MaterialsSetup

Updates the IsActive for the specified MaterialsSetup.

The following elements are required:

company (string) The company for the Materials Setup.
value (bool) In the request body, supply the new value for the IsActive.

Response:

If successful, this method returns an empty response body.

Response status codes:

200 The operation was successful.

Resource URL:

PUT /materialsCore/materialsSetups/{company}/isActive

PUT Set NegativeStock MaterialsSetup

Updates the NegativeStock for the specified Materials Setup.

The following elements are required:

company (string) The company for the Materials Setup.
value (bool) In the request body, supply the new value for the NegativeStock.

Response:

If successful, this method returns an empty response body.

Response status codes:

200 The operation was successful.

Resource URL:

PUT /materialsCore/materialsSetups/{company}/negativeStock
