Financials | Account Transaction

Overview

URL api/{tenantKey}/{orgKey}/financialCore/accountTransactions
Methods Supported GET, POST, PUT, DELETE
Description Service to manage entities of type 'AccountTransaction'.

GET OPERATIONS

GET AccountTransactions

Returns all Account Transaction user has access to.

Response:

If successful, this method returns a list of Account Transaction. See AccountTransactionResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /financialCore/accountTransactions

GET AccountTransactions (paged)

Returns a paginated list of Account Transaction user has access to.

The following elements are optional:

page (int) The number of the page you want to consume the results.
pageSize (int) Specify how many items you want each page to return, up to 1000 items.

Response:

If successful, this method returns a list of Account Transaction. See AccountTransactionResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /financialCore/accountTransactions

GET AccountTransaction by identifier

Returns information for a given Account Transaction, by specifying its identifier.

The following elements are required:

id (string) The identifier for the Account Transaction.

Response:

If successful, returns a resource of the Account Transaction. See AccountTransactionResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /financialCore/accountTransactions/{id}

GET AccountTransaction by key

Returns information for a given Account Transaction, by specifying its key.

The following elements are required:

Response:

Response status codes:

200 The operation was successful.

Resource URL:

GET /financialCore/accountTransactions/{accountTransactionKey}

GET Print AccountTransaction by identifier

Prints the specified AccountTransaction.

The following elements are required:

The following elements are optional:

POST OPERATIONS

POST AccountTransactions

Creates a new Account Transaction.

The following elements are required:

accountTransactionResource (ApiAccountTransactionResource) In the request body, supply a resource containing the Account Transaction information.
See ApiAccountTransactionResource.

Response:

If successful, returns the Account Transaction identifier in the response body and the URL in the response header Location property, or an error message. See ApiAccountTransactionResource.

Response status codes:

201 The operation was successful and the Account Transaction was created.

Resource URL:

POST /financialCore/accountTransactions
