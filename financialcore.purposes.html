Financials | Purpose

Overview

URL api/{tenantKey}/{orgKey}/financialCore/purposes
Methods Supported GET, POST, PUT, DELETE
Description Service to manage entities of type 'Purpose'.

GET OPERATIONS

GET Purposes

Returns all Purpose user has access to.

Response:

If successful, this method returns a list of Purpose. See PurposeResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /financialCore/purposes

GET Purposes (paged)

Returns a paginated list of Purpose user has access to.

The following elements are optional:

page (int) The number of the page you want to consume the results.
pageSize (int) Specify how many items you want each page to return, up to 1000 items.

Response:

If successful, this method returns a list of Purpose. See PurposeResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /financialCore/purposes

GET Purpose by identifier

Returns information for a given Purpose, by specifying its identifier.

The following elements are required:

id (string) The identifier for the Purpose.

Response:

If successful, returns a resource of the Purpose. See PurposeResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /financialCore/purposes/{id}

GET Purpose by key

Returns information for a given Purpose, by specifying its key.

The following elements are required:

purposeKey (string) Purpose

Response:

If successful, returns a resource of the Purpose. See PurposeResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /financialCore/purposes/{purposeKey}

GET Print Purpose by identifier

Prints the specified Purpose.

The following elements are required:

id (string) The identifier for the Purpose.

The following elements are optional:

template (string) The report template.

Response:

If successful, returns a stream with the file of the printed Purpose.

Response status codes:

200 The operation was successful.

Resource URL:

GET /financialCore/purposes/{id}/print

GET Print Purpose by key

Prints the specified Purpose by specifying its key.

The following elements are required:

The following elements are optional:

Resource URL:

GET /financialCore/purposes/{purposeKey}/print

POST OPERATIONS

POST Purposes

Creates a new Purpose.

The following elements are required:

purposeResource (ApiPurposeResource) In the request body, supply a resource containing the Purpose information.
See ApiPurposeResource.

Response:

If successful, returns the Purpose identifier in the response body and the URL in the response header Location property, or an error message. See ApiPurposeResource.

Response status codes:

201 The operation was successful and the Purpose was created.

Resource URL:

POST /financialCore/purposes
