Overview

URL api/{tenantKey}/{orgKey}/corePatterns/contacts
Methods Supported GET, POST, PUT, DELETE
Description Service to manage entities of type 'Contact'.

GET OPERATIONS

GET Contacts

Returns all Contact user has access to.

Response:

If successful, this method returns a list of Contact. See ContactResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /corePatterns/contacts

GET Contacts (paged)

Returns a paginated list of Contact user has access to.

The following elements are optional:

page (int) The number of the page you want to consume the results.
pageSize (int) Specify how many items you want each page to return, up to 1000 items.

Response:

If successful, this method returns a list of Contact. See ContactResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /corePatterns/contacts

GET Contact by identifier

Returns information for a given Contact, by specifying its identifier.

The following elements are required:

id (string) The identifier for the Contact.

Response:

If successful, returns a resource of the Contact. See ContactResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /corePatterns/contacts/{id}

GET Contact by key

Returns information for a given Contact, by specifying its key.

The following elements are required:

fileNumber (int) Defines the key of a contact. This attribute is calculated by the system.

Response:

If successful, returns a resource of the Contact. See ContactResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /corePatterns/contacts/{fileNumber}

GET Print Contact by identifier

Prints the specified Contact.

The following elements are required:

The following elements are optional:

template (string) The report template.

Response:

If successful, returns a stream with the file of the printed Contact.

Response status codes:

200 The operation was successful.

Resource URL:

GET /corePatterns/contacts/{id}/print

Response:

Response status codes:

GET Contact Picture.

Returns the Picture of the specified Contact. This method returns an image stream.

The following elements are required:

fileName (string) File name.

POST OPERATIONS

POST Contacts

Creates a new Contact.

The following elements are required:

contactResource (ApiContactResource) In the request body, supply a resource containing the Contact information.
See ApiContactResource.

Response:

If successful, returns the Contact identifier in the response body and the URL in the response header Location property, or an error message. See ApiContactResource.

Response status codes:

201 The operation was successful and the Contact was created.

Resource URL:

POST /corePatterns/contacts

Resources


ContactResource

The following elements are required:

fileNumber (int) Defines the key of a contact. This attribute is calculated by the system.
name (string) The name of the contact.
contactType (string) The contact type.

The following elements are optional:

greeting (string) The phrase used to address a contact party in any correspondence.
jobTitle (string) The free form text for job title.
title (string) The title or a salutary introduction for a contact, such as Mr., Herr, Miss, and so on.
firstName (string) The first name of the person.
middleName (string) The middle name of the person.
lastName (string) The last name of the person.
electronicMail (string) The e-mail address of the contact point.
websiteUrl (string) The website address of the contact.
businessTelephone (string) The business telephone number formatted in the local format, including the area code, country code, and extension.
homeTelephone (string) The home phone number formatted in the local format, including the area code, country code, and extension.
mobile (string) The mobile phone number formatted in the local format, including the area code, country code, and extension.
facebook (string) The contact facebook url.
linkedIn (string) The contact LinkedIn url.
notes (string) Free text note associated with the contact.
picture (string) The contact picture.
birthDate (DateTime) The contact birth date.
streetName (string) The contact street name.
buildingNumber (string) The contact building number.
cityName (string) The contact city name.
postalZone (string) The contact postal zone.
country (string) The contact country.
address (string) The contact address.
maritalStatus (enum MaritalStatus) The contact marital status.
See MaritalStatus.
gender (enum Gender) The contact gender.
See Gender.
partyId (Guid) The contact associated party.
belongsToParty (bool) Indicates if the contact is associated to a party.
