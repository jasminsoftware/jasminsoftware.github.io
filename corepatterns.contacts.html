GET Operations

GET Contacts

GET Contacts

Returns all Contact user has access to.

Response:

If successful, this method returns a list of Contact. See ContactResource.

Response status codes:

200 The operation was successful.

Resource URL:

api/{tenantKey}/{orgKey}/corePatterns/contacts
GET /corePatterns/contacts

GET Contacts (paged)

GET Contacts (paged)

Returns a paginated list of Contact user has access to.

The following elements are optional:

page (int) The number of the page you want to consume the results. This is a query string parameter.
pageSize (int) Specify how many items you want each page to return, up to 1000 items. This is a query string parameter.

Response:

If successful, this method returns a list of Contact. See ContactResource.

Response status codes:

200 The operation was successful.

Resource URL:

api/{tenantKey}/{orgKey}/corePatterns/contacts
GET /corePatterns/contacts

GET Contact by identifier

POST Operations

POST Contacts

POST Contacts

Creates a new Contact.

The following elements are required:

contactResource (ApiContactResource) In the request body, supply a resource containing the Contact information.
See ApiContactResource.

Response:

If successful, returns the Contact identifier in the response body and the URL in the response header Location property, or an error message. See ApiContactResource.

Response status codes:

201 The operation was successful and the Contact was created.

Resource URL:

api/{tenantKey}/{orgKey}/corePatterns/contacts
POST /corePatterns/contacts
