URL api/{tenantKey}/{orgKey}/operations/notifications
Methods Supported GET, POST, PUT, DELETE
Description Service to manage entities of type 'Notification'.

GET OPERATIONS

GET Notifications

Returns all Notification user has access to.

Response:

If successful, this method returns a list of Notification. See NotificationResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /operations/notifications

GET Notifications (paged)

Returns a paginated list of Notification user has access to.

The following elements are optional:

page (int) The number of the page you want to consume the results.
pageSize (int) Specify how many items you want each page to return, up to 1000 items.

Response:

If successful, this method returns a list of Notification. See NotificationResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /operations/notifications

GET Notification by identifier

Returns information for a given Notification, by specifying its identifier.

The following elements are required:

POST OPERATIONS

POST Notifications

Creates a new Notification.

The following elements are required:

notificationResource (ApiNotificationResource) In the request body, supply a resource containing the Notification information.
See ApiNotificationResource.

Response:

If successful, returns the Notification identifier in the response body and the URL in the response header Location property, or an error message. See ApiNotificationResource.

Response status codes:

201 The operation was successful and the Notification was created.

Resource URL:

POST /operations/notifications
