GET OPERATIONS

GET InvoiceTypes

Returns all Invoice Type user has access to.

Response:

If successful, this method returns a list of Invoice Type. See InvoiceTypeResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /salesCore/invoiceTypes

Returns a paginated list of Invoice Type user has access to.

The following elements are optional:

POST OPERATIONS

POST InvoiceTypes

Creates a new Invoice Type.

The following elements are required:

invoiceTypeResource (ApiInvoiceTypeResource) In the request body, supply a resource containing the Invoice Type information.
See ApiInvoiceTypeResource.

Response:

If successful, returns the Invoice Type identifier in the response body and the URL in the response header Location property, or an error message. See ApiInvoiceTypeResource.

Response status codes:

201 The operation was successful and the Invoice Type was created.

Resource URL:

POST /salesCore/invoiceTypes

POST Invoice Type

Inserts a new Invoice Type Serie in an existing Invoice Type.

The following elements are required:

typeKey (string) The invoice type key.
companyKey (string) Company key to which the requested Invoice Type is associated to.
value (ApiInvoiceTypeDocumentTypeSeriesResource) The ApiInvoiceTypeDocumentTypeSeriesResource instance.
See ApiInvoiceTypeDocumentTypeSeriesResource.

Response:

If successful, this method returns an empty response body.

Response status codes:

204 The operation was successful.

Resource URL:

POST /salesCore/invoiceTypes/{companyKey}/{typeKey}/documentTypeSeries

Resources


InvoiceTypeResource

The following elements are required:

company (string) The invoice type company. The company code is the central organizational unit within the system.
typeKey (string) The invoice type key.
cashInvoice (bool) It is an invoice which also documents an on-time payment receipt in cash, unlike an invoice or a promise of payment.
accountingSchema (enum AccountingSchema) The entity's accounting organization method. This organization can follow the rules of "Cash" or "Accrual".
See AccountingSchema.
isPortugueseCompany (bool) Indicates if the invoice type is targeted at a company that is portuguese.

The following elements are optional:

description (string) The invoice type description.
documentTypeSeries (List of InvoiceTypeSerieResource) The invoice type details.
See InvoiceTypeSerieResource.
fiscalDocumentType (string) The fiscal document type.
defaultPettyCash (string) It refers to small amounts of cash kept in a business' cash drawer. There are several reasons for having petty cash:- It can be used as small change for customers or patients; It can be used to pay for small purchases that require cash, such as food for having lunch in the office, coffee supplies, or parking.Each purchase made with the money kept in the cash drawer should be documented, in much the same as other business revenues or expenses. When the amount in the drawer is too large, a deposit voucher should be created to transfer the funds to the bank.
defaultCashFlowItem (string) Type of cash flow item to be used when CashInvoice is active.
defaultCreditMemoType (string) The type of credit memo type for the invoice type.
defaultDebitMemoType (string) The type of debit memo type for the invoice type.
contactType (string) The contact type for the invoice type.
notification (string) The default notification for documents with this invoice type.
