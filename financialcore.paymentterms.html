Financials | Payment Term

Overview

URL api/{tenantKey}/{orgKey}/financialCore/paymentTerms
Methods Supported GET, POST, PUT, DELETE
Description Service to manage entities of type 'PaymentTerm'.

GET OPERATIONS

GET PaymentTerms

Returns all Payment Term user has access to.

Response:

If successful, this method returns a list of Payment Term. See PaymentTermResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /financialCore/paymentTerms

GET PaymentTerms (paged)

Returns a paginated list of Payment Term user has access to.

The following elements are optional:

page (int) The number of the page you want to consume the results.
pageSize (int) Specify how many items you want each page to return, up to 1000 items.

Response:

If successful, this method returns a list of Payment Term. See PaymentTermResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /financialCore/paymentTerms

GET PaymentTerm by identifier

Returns information for a given Payment Term, by specifying its identifier.

The following elements are required:

POST OPERATIONS

POST PaymentTerms

Creates a new Payment Term.

The following elements are required:

paymentTermResource (ApiPaymentTermResource) In the request body, supply a resource containing the Payment Term information.
See ApiPaymentTermResource.

Response:

If successful, returns the Payment Term identifier in the response body and the URL in the response header Location property, or an error message. See ApiPaymentTermResource.

Response status codes:

201 The operation was successful and the Payment Term was created.

Resource URL:

POST /financialCore/paymentTerms

Resources


PaymentTermResource

The following elements are required:

paymentTermKey (string) Payment term key
description (string) Payment term description
useInAccountsReceivable (bool) Indicates whether this payment term can be used in accounts receivable or not
useInAccountsPayable (bool) Indicates whether this payment term can be used in accounts payable or not

The following elements are optional:

validFrom (DateTime) Start date for validity period
validTo (DateTime) End date for validity period
daysFromReferenceDate (int) Number of days that determine the term for this payment
