Platform | Culture

Overview

URL api/{tenantKey}/{orgKey}/corePatterns/cultures
Methods Supported GET, POST, PUT, DELETE
Description Service to manage entities of type 'Culture'.

GET OPERATIONS

GET Cultures

Returns all Culture user has access to.

Response:

If successful, this method returns a list of Culture. See CultureResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /corePatterns/cultures

GET Cultures (paged)

Returns a paginated list of Culture user has access to.

The following elements are optional:

page (int) The number of the page you want to consume the results.
pageSize (int) Specify how many items you want each page to return, up to 1000 items.

Response:

If successful, this method returns a list of Culture. See CultureResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /corePatterns/cultures

GET Culture by identifier

Returns information for a given Culture, by specifying its identifier.

The following elements are required:

id (string) The identifier for the Culture.

Response:

If successful, returns a resource of the Culture. See CultureResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /corePatterns/cultures/{id}

GET Culture by key

Returns information for a given Culture, by specifying its key.

The following elements are required:

cultureKey (string) The culture key.

Response:

If successful, returns a resource of the Culture. See CultureResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /corePatterns/cultures/{cultureKey}

GET Print Culture by identifier

Prints the specified Culture.

The following elements are required:

id (string) The identifier for the Culture.

The following elements are optional:

template (string) The report template.

Response:

If successful, returns a stream with the file of the printed Culture.

Response status codes:

200 The operation was successful.

Resource URL:

GET /corePatterns/cultures/{id}/print

GET Print Culture by key

Prints the specified Culture by specifying its key.

The following elements are required:

cultureKey (string) The culture key.

The following elements are optional:

template (string) The report template.

Response:

If successful, returns a stream with the file of the printed Culture.

Response status codes:

200 The operation was successful.

Resource URL:

GET /corePatterns/cultures/{cultureKey}/print

POST OPERATIONS

POST Cultures

Creates a new Culture.

The following elements are required:

cultureResource (ApiCultureResource) In the request body, supply a resource containing the Culture information.
See ApiCultureResource.

Response:

If successful, returns the Culture identifier in the response body and the URL in the response header Location property, or an error message. See ApiCultureResource.

Response status codes:

201 The operation was successful and the Culture was created.

Resource URL:

POST /corePatterns/cultures

PUT OPERATIONS

The following elements are required:
