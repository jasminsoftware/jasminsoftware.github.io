Materials Management | Item Subtype

Overview

URL api/{tenantKey}/{orgKey}/materialsCore/itemSubtypes
Methods Supported GET, POST, PUT, DELETE
Description Service to manage entities of type 'ItemSubType'.

GET OPERATIONS

GET ItemSubtypes

Returns all Item Subtype user has access to.

Response:

If successful, this method returns a list of Item Subtype. See ItemSubtypeResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /materialsCore/itemSubtypes

GET ItemSubtypes (paged)

Returns a paginated list of Item Subtype user has access to.

The following elements are optional:

page (int) The number of the page you want to consume the results.
pageSize (int) Specify how many items you want each page to return, up to 1000 items.

Response:

If successful, this method returns a list of Item Subtype. See ItemSubtypeResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /materialsCore/itemSubtypes

POST OPERATIONS

POST ItemSubtypes

Creates a new Item Subtype.

The following elements are required:

itemSubtypeResource (ApiItemSubtypeResource) In the request body, supply a resource containing the Item Subtype information.
See ApiItemSubtypeResource.

Response:

If successful, returns the Item Subtype identifier in the response body and the URL in the response header Location property, or an error message. See ApiItemSubtypeResource.

Response status codes:

201 The operation was successful and the Item Subtype was created.

Resource URL:

POST /materialsCore/itemSubtypes
