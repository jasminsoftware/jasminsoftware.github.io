Materials Management | Warehouse

Overview

URL api/{tenantKey}/{orgKey}/materialsCore/warehouses
Methods Supported GET, POST, PUT, DELETE
Description Service to manage entities of type 'Warehouse'.

GET OPERATIONS

GET Warehouses

Returns all Warehouse user has access to.

Response:

If successful, this method returns a list of Warehouse. See WarehouseResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /materialsCore/warehouses

GET Warehouses (paged)

Returns a paginated list of Warehouse user has access to.

The following elements are optional:

page (int) The number of the page you want to consume the results.
pageSize (int) Specify how many items you want each page to return, up to 1000 items.

Response:

If successful, this method returns a list of Warehouse. See WarehouseResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /materialsCore/warehouses

GET Warehouse by identifier

Returns information for a given Warehouse, by specifying its identifier.

The following elements are required:

id (string) The identifier for the Warehouse.

Response:

If successful, returns a resource of the Warehouse. See WarehouseResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /materialsCore/warehouses/{id}

GET Warehouse by key

Returns information for a given Warehouse, by specifying its key.

The following elements are required:

warehouseKey (string) The key for the Warehouse.

Response:

If successful, returns a resource of the Warehouse. See WarehouseResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /materialsCore/warehouses/{warehouseKey}

GET Print Warehouse by identifier

Prints the specified Warehouse.

The following elements are required:

id (string) The identifier for the Warehouse.

The following elements are optional:

template (string) The report template.

Response:

If successful, returns a stream with the file of the printed Warehouse.

Response status codes:

200 The operation was successful.

Resource URL:

GET /materialsCore/warehouses/{id}/print

The following elements are required:

Resource URL:

POST OPERATIONS

POST Warehouses

Creates a new Warehouse.

The following elements are required:

warehouseResource (ApiWarehouseResource) In the request body, supply a resource containing the Warehouse information.
See ApiWarehouseResource.

Response:

If successful, returns the Warehouse identifier in the response body and the URL in the response header Location property, or an error message. See ApiWarehouseResource.

Response status codes:

201 The operation was successful and the Warehouse was created.

Resource URL:

POST /materialsCore/warehouses

Resources


WarehouseResource

The following elements are required:

warehouseKey (string) The key for the Warehouse.
company (string) The company for the Warehouse.
allowItemEntries (bool) Determines if this warehouse accepts item entries.
allowItemIssues (bool) Determines if this warehouse accepts item issues.
isExternallyManaged (bool) Indicates whether the warehouse is externally managed.
isLocked (bool) Indicates if the warehouse is currently locked.

The following elements are optional:

description (string) The description for the Warehouse.
streetName (string) The street name for the Warehouse.
buildingNumber (string) The building number for the Warehouse.
cityName (string) The city name for the Warehouse.
postalZone (string) The postal zone for the Warehouse.
closedExercise (DateTime) The closed exercose date for the Warehouse.
externalId (string) The external identification for the Warehouse.
externalVersion (string) The external version for the Warehouse.
region (string) The region for the Warehouse.
country (string) The country for the Warehouse.
