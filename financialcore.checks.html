Financials | Check

Overview

URL api/{tenantKey}/{orgKey}/financialCore/checks
Methods Supported GET, POST, PUT, DELETE
Description Service to manage entities of type 'Check'.

GET OPERATIONS

GET Checks

Returns all Check user has access to.

Response:

If successful, this method returns a list of Check. See CheckResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /financialCore/checks

GET Checks (paged)

Returns a paginated list of Check user has access to.

The following elements are optional:

page (int) The number of the page you want to consume the results.
pageSize (int) Specify how many items you want each page to return, up to 1000 items.

Response:

If successful, this method returns a list of Check. See CheckResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /financialCore/checks

GET Check by identifier

Returns information for a given Check, by specifying its identifier.

The following elements are required:

id (string) The identifier for the Check.

Response:

If successful, returns a resource of the Check. See CheckResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /financialCore/checks/{id}

GET Check by key

Returns information for a given Check, by specifying its key.

The following elements are required:

companyKey (string) Company key to which the requested Check is associated to.
financialInstitution (string) Financial Institution
financialAccount (string) Bank Account
checkNumber (string) Check Number

Response:

If successful, returns a resource of the Check. See CheckResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /financialCore/checks/{companyKey}/{financialInstitution}/{financialAccount}/{checkNumber}

GET Print Check by identifier

Prints the specified Check.

The following elements are required:

id (string) The identifier for the Check.

The following elements are optional:

template (string) The report template.

Response:

If successful, returns a stream with the file of the printed Check.

Response status codes:

200 The operation was successful.

Resource URL:

GET /financialCore/checks/{id}/print

GET Print Check by key

Prints the specified Check by specifying its key.

The following elements are required:

companyKey (string) Company key to which the requested Check is associated to.
financialInstitution (string) Financial Institution
financialAccount (string) Bank Account
checkNumber (string) Check Number

The following elements are optional:

Response:

Response status codes:

200 The operation was successful.

Resource URL:

GET /financialCore/checks/{companyKey}/{financialInstitution}/{financialAccount}/{checkNumber}/print

POST OPERATIONS

POST Checks

Creates a new Check.

The following elements are required:

checkResource (ApiCheckResource) In the request body, supply a resource containing the Check information.
See ApiCheckResource.

Response:

If successful, returns the Check identifier in the response body and the URL in the response header Location property, or an error message. See ApiCheckResource.

Response status codes:

201 The operation was successful and the Check was created.

Resource URL:

POST /financialCore/checks
