Overview

URL api/{tenantKey}/{orgKey}/corePatterns/operations
Methods Supported GET
Description Service to manage entities of type 'Operation'.

GET OPERATIONS

GET Operations

Returns all of the Operations user has access to.

Response:

If successful, this method returns a list of Operations. See OperationResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /corePatterns/operations

GET Operations (paged)

Returns a paginated list of Operations user has access to.

The following elements are optional:

page (int) The number of the page you want to consume the results.
pageSize (int) Specify how many items you want each page to return, up to 1000 items.

Response:

If successful, this method returns a list of Operations. See OperationResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /corePatterns/operations

GET Operation by identifier

Returns information for the specified Operation, by specifying its identifier.

The following elements are required:

id (string) The identifier for the Operation.

Response:

If successful, returns a resource of the Operation. See OperationResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /corePatterns/operations/{id}

GET Operation by key

Returns information for the specified Operation, by specifying its key.

The following elements are required:

name (string) Name
module (string) Module
serviceName (string) Service Name

Response:

If successful, returns a resource of the Operation. See OperationResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /corePatterns/operations/{name}/{module}/{serviceName}
