GET Operations

Returns all Operation user has access to.

Response:

If successful, this method returns a list of Operation. See OperationResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /corePatterns/operations

GET Operations (paged)

GET Operations (paged)

Returns a paginated list of Operation user has access to.

The following elements are optional:

page (int) The number of the page you want to consume the results. This is a query string parameter.
pageSize (int) Specify how many items you want each page to return, up to 1000 items. This is a query string parameter.

Response:

If successful, this method returns a list of Operation. See OperationResource.

Response status codes:

200 The operation was successful.

Resource URL:

api/{tenantKey}/{orgKey}/corePatterns/operations
GET /corePatterns/operations

GET Operation by identifier

GET Operation by identifier

Returns information for a given Operation, by specifying its identifier.

The following elements are required:

id (string) The identifier for the Operation.

Response:

If successful, returns a resource of the Operation. See OperationResource.

Response status codes:

200 The operation was successful.

Resource URL:

api/{tenantKey}/{orgKey}/corePatterns/operations/{id}
GET /corePatterns/operations/{id}

GET Operation by key

GET Operation by key

Returns information for a given Operation, by specifying its key.

The following elements are required:

module (string) The operation’s module.
serviceName (string) The operation’s service name.

Response:

If successful, returns a resource of the Operation. See OperationResource.

Response status codes:
