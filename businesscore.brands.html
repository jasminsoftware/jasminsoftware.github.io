Master Data Business Entities | Brand

Overview

URL api/{tenantKey}/{orgKey}/businessCore/brands
Methods Supported GET, POST, PUT, DELETE
Description Service to manage entities of type 'Brand'. The trade name used by a company which identifies its products.

GET OPERATIONS

GET Brands

Returns all Brand user has access to.

Response:

If successful, this method returns a list of Brand. See BrandResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /businessCore/brands

GET Brands (paged)

Returns a paginated list of Brand user has access to.

The following elements are optional:

page (int) The number of the page you want to consume the results.
pageSize (int) Specify how many items you want each page to return, up to 1000 items.

Response:

If successful, this method returns a list of Brand. See BrandResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /businessCore/brands

GET Brand by identifier

Returns information for a given Brand, by specifying its identifier.

The following elements are required:

id (string) The identifier for the Brand.

Response:

If successful, returns a resource of the Brand. See BrandResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /businessCore/brands/{id}

GET Brand by key

Returns information for a given Brand, by specifying its key.

The following elements are required:

brandKey (string) The key for the item brand.

Response:

If successful, returns a resource of the Brand. See BrandResource.

Response status codes:

200 The operation was successful.

Resource URL:

GET /businessCore/brands/{brandKey}

GET Print Brand by identifier

Prints the specified Brand.

The following elements are required:

id (string) The identifier for the Brand.

The following elements are optional:

template (string) The report template.

Response:

If successful, returns a stream with the file of the printed Brand.

Response status codes:

200 The operation was successful.

Resource URL:

GET /businessCore/brands/{id}/print

GET Print Brand by key

Prints the specified Brand by specifying its key.

The following elements are required:

The following elements are optional:

Response:

Response status codes:

200 The operation was successful.

Resource URL:

GET /businessCore/brands/{brandKey}/print

GET Brand Picture.

Returns the Picture of the specified Brand. This method returns an image stream.

The following elements are required:

id (string) The identifier for the Brand.
fileName (string) File name.

Response:

If successful, returns a stream with the image file of the Brand Picture.

Response status codes:

200 The operation was successful.

Resource URL:

GET /businessCore/brands/{id}/picture

POST OPERATIONS

POST Brands

Creates a new Brand.

The following elements are required:

brandResource (ApiBrandResource) In the request body, supply a resource containing the Brand information.
See ApiBrandResource.

Response:

If successful, returns the Brand identifier in the response body and the URL in the response header Location property, or an error message. See ApiBrandResource.

Response status codes:

201 The operation was successful and the Brand was created.

Resource URL:

POST /businessCore/brands
